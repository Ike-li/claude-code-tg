## 二、经验证的核心发现

### 发现 1：Marvis 无 CLI/API 集成 ✅ 高可信度

**验证状态**: 1-0 票通过（1 票确认，0 票反对，2 票因速率限制未完成）

**证据**:
- 官网 (marvis.qq.com) 无 CLI 或 API 文档
- 中英文全网搜索均无相关文档
- 产品定位为 GUI 桌面助手

**引用**:
> "Product documentation shows Marvis is a GUI-based desktop assistant with mobile app control. No mention of CLI, API endpoints, or programmatic access for external bot integration like Telegram."

**影响**:
这是 Marvis 无法替代 tgcc 的**致命伤**。tgcc 架构依赖 Claude Code CLI，而 Marvis 无编程接口。

---

### 发现 2：系统级控制能力 ✅ 高可信度

**验证状态**: 2-0 票通过（2 票确认，0 票反对，1 票未完成）

**证据**:
- 官网明确说明："一句话完成电脑设置"
- 可查询硬件配置、电池健康、网络状态
- 可修改系统设置、优化启动项、清理文件

**引用**:
> "Official site documents 'one-sentence commands to check hardware configuration and modify system settings.'"

**影响**:
这是 Marvis **超越** tgcc 的领域。tgcc 无系统级权限。

---

### 发现 3：双模式运行 ⚠️ 中等可信度

**验证状态**: 1-1 票（1 票确认，1 票反对，1 票未完成）

**证据**:
- 官网确认："效率模式"和"隐私模式"
- 36kr 评测文章提及使用混元、DeepSeek V4（效率模式）和 Qwen（隐私模式）
- 但技术细节来自二手来源

**引用**:
> "36kr article explicitly states cloud efficiency mode uses 'Hunyuan and DeepSeek V4 models' while local privacy mode uses 'Qwen edge-side model'."

**争议**:
具体模型选择的准确性未经官方技术文档确认。

---

### 发现 4：远程控制功能争议 ⚠️ 存在严重争议

**验证状态**: 1-1 票（1 票确认，1 票反对，1 票未完成）

**官方声称**:
- "手机连接电脑，实时查看任务执行画面"
- "手机随时操控电脑"

**验证发现的问题**:
1. **移动 App 未上线**
   - Google Play / App Store **搜索不到** Marvis 移动 App
   - iOS 标记为"送审中，6月中旬上线"（调研时仍未上线）

2. **"远程解锁 PC" 被证伪**
   - 验证投票：0-1（被反驳）
   - 理由：违反标准安全实践

**结论**:
官网功能声称与实际可用性存在差距。这是**营销夸大**的典型案例。

---

### 发现 5：CodeBuddy 是独立产品 ✅ 高可信度

**验证状态**: 1-1 票通过

**关键区分**:
- **Marvis**: 系统助手（marvis.qq.com）
- **CodeBuddy**: IDE 代码助手（copilot.tencent.com）

**CodeBuddy 的能力**（不属于 Marvis）:
- Subagent 配置（System Prompts、Tools、MCP 服务）
- Agentic 模式和 Manual 模式
- 项目级和用户级作用域

**重要警告**:
调研中发现多处将 CodeBuddy 的功能误归于 Marvis。**它们是两个完全独立的产品**。

## 六、核心结论

### ❌ Marvis 无法替代 tgcc

**已验证的原因**（高可信度）:

1. **无 CLI/API 集成**（验证通过）
   - Marvis 是 GUI 桌面助手
   - 无法被外部程序调用
   - tgcc 需要 CLI 接口

2. **架构根本不兼容**（工作流明确）
   - Marvis: 移动 App(?) ↔ 桌面端 ↔ 操作系统 API
   - tgcc: Telegram ↔ Python 服务 ↔ Claude Code CLI ↔ 项目目录
   - 两者通信模式完全不同

3. **生态隔离**（逻辑推导）
   - Marvis 独立生态，与 Claude 无关
   - tgcc 深度依赖 Claude Code 生态

**部分验证的原因**（中等可信度）:

4. **移动端不成熟**（存在争议）
   - 官网声称支持，但 App 未在应用商店上线
   - iOS 送审中未上线

## 七、使用场景建议

### 选择 tgcc ✅

**适用场景**:
- 核心需求是**远程代码开发**
- 需要 Claude Code 完整能力（上下文、技能、多文件编辑）
- 管理多个独立项目
- 灵活权限控制
- 开源、可自定义

**数据支持**: CLI/API 缺失已验证（1-0 票）

---

### 选择 Marvis ✅

**适用场景**:
- 需要**系统级管理**（已验证：2-0 票）
- 文件智能分类
- 跨应用自动化
- 不需要远程编程
- 接受闭源产品

**注意**: 移动端功能存在争议，建议等 App 正式上线后再评估

---

### 同时使用 🔄

**互补方案**:
- Marvis: 系统管理、文件整理
- tgcc: 代码开发、项目管理

**可行性**: 两者不冲突，可共存
